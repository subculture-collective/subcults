# Testing Guide

How to write, run, and debug tests across the Subcults codebase.

## Quick Reference

```bash
make test             # Go + frontend tests (race detector enabled)
make test-coverage    # Generate HTML coverage reports
make test-integration # Integration tests (requires Docker)
make test-e2e         # Playwright E2E tests
make test-load        # k6 load testing
make test-all         # Everything
```

## Coverage Targets

| Scope | Target | Enforcement |
|-------|--------|-------------|
| Backend overall | >80% | CI gate via Codecov |
| Frontend overall | >70% | CI gate via Codecov |
| Critical packages (auth, geo, payment) | 95% | Codecov per-package threshold |
| Handlers | >80% | Codecov per-package |
| Repositories | >85% | Codecov per-package |
| New code (patch) | >70% | Codecov patch check |

Coverage reports are generated by CI and posted to PRs. View locally:

```bash
make test-coverage
# Backend: open coverage.html
# Frontend: open web/coverage/index.html
```

## Backend Testing (Go)

### Running Tests

```bash
# All backend tests with race detection
go test -race ./...

# Specific package
go test -race ./internal/auth/...

# Specific test
go test -race -run TestCreateScene_Success ./internal/api/...

# With coverage
go test -race -coverprofile=coverage.out ./...
go tool cover -html=coverage.out -o coverage.html

# Verbose output
go test -v -race -run TestJitter ./internal/geo/...
```

### Writing Unit Tests

Use table-driven tests with descriptive subtest names:

```go
func TestValidateSceneName(t *testing.T) {
    tests := []struct {
        name    string
        input   string
        wantErr bool
    }{
        {name: "valid name", input: "Underground Club", wantErr: false},
        {name: "empty name", input: "", wantErr: true},
        {name: "too long", input: strings.Repeat("x", 201), wantErr: true},
        {name: "whitespace only", input: "   ", wantErr: true},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            errs := validate.SceneName(tt.input)
            if (len(errs) > 0) != tt.wantErr {
                t.Errorf("SceneName(%q) errors = %v, wantErr = %v", tt.input, errs, tt.wantErr)
            }
        })
    }
}
```

### Testing HTTP Handlers

Use `httptest` with in-memory repositories:

```go
func TestHandleGetScene_Success(t *testing.T) {
    // Setup: create an in-memory repository with test data
    repo := scene.NewInMemorySceneRepository()
    testScene := &scene.Scene{
        ID:   "scene-123",
        Name: "Test Venue",
    }
    repo.Create(context.Background(), testScene)

    // Create request
    req := httptest.NewRequest("GET", "/api/scenes/scene-123", nil)
    w := httptest.NewRecorder()

    // Execute handler
    handler := api.HandleGetScene(repo)
    handler.ServeHTTP(w, req)

    // Assert
    if w.Code != http.StatusOK {
        t.Fatalf("expected 200, got %d: %s", w.Code, w.Body.String())
    }

    var got scene.Scene
    json.NewDecoder(w.Body).Decode(&got)
    if got.Name != "Test Venue" {
        t.Errorf("expected name 'Test Venue', got %q", got.Name)
    }
}

func TestHandleGetScene_NotFound(t *testing.T) {
    repo := scene.NewInMemorySceneRepository()

    req := httptest.NewRequest("GET", "/api/scenes/nonexistent", nil)
    w := httptest.NewRecorder()

    handler := api.HandleGetScene(repo)
    handler.ServeHTTP(w, req)

    if w.Code != http.StatusNotFound {
        t.Errorf("expected 404, got %d", w.Code)
    }
}
```

### Testing with Authentication Context

Inject user identity into the request context:

```go
func TestHandleCreateScene_Authenticated(t *testing.T) {
    repo := scene.NewInMemorySceneRepository()
    handler := api.HandleCreateScene(repo)

    body := `{"name": "My Scene", "description": "test"}`
    req := httptest.NewRequest("POST", "/api/scenes", strings.NewReader(body))
    req.Header.Set("Content-Type", "application/json")

    // Inject authenticated user
    ctx := middleware.SetUserDID(req.Context(), "did:plc:testuser123")
    req = req.WithContext(ctx)

    w := httptest.NewRecorder()
    handler.ServeHTTP(w, req)

    if w.Code != http.StatusCreated {
        t.Fatalf("expected 201, got %d: %s", w.Code, w.Body.String())
    }
}
```

### Testing Middleware

Use a pipeline of middleware wrapping a test handler:

```go
func TestLogging_BasicFields(t *testing.T) {
    var buf bytes.Buffer
    logger := slog.New(slog.NewJSONHandler(&buf, &slog.HandlerOptions{Level: slog.LevelDebug}))

    handler := middleware.Logging(logger)(http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        w.WriteHeader(http.StatusOK)
    }))

    req := httptest.NewRequest("GET", "/api/scenes", nil)
    w := httptest.NewRecorder()
    handler.ServeHTTP(w, req)

    // Parse structured log output
    var logEntry map[string]interface{}
    json.Unmarshal(buf.Bytes(), &logEntry)

    if logEntry["method"] != "GET" {
        t.Errorf("expected method GET, got %v", logEntry["method"])
    }
    if logEntry["path"] != "/api/scenes" {
        t.Errorf("expected path /api/scenes, got %v", logEntry["path"])
    }
}
```

### Testing Privacy Invariants

Privacy enforcement tests are mandatory for any code touching location data:

```go
func TestScene_EnforceLocationConsent(t *testing.T) {
    tests := []struct {
        name         string
        allowPrecise bool
        hasPoint     bool
        wantPoint    bool
    }{
        {"precise allowed with point", true, true, true},
        {"precise denied clears point", false, true, false},
        {"precise denied without point", false, false, false},
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            s := &scene.Scene{AllowPrecise: tt.allowPrecise}
            if tt.hasPoint {
                s.PrecisePoint = &scene.Point{Lat: 40.7128, Lng: -74.0060}
            }
            s.EnforceLocationConsent()
            if (s.PrecisePoint != nil) != tt.wantPoint {
                t.Errorf("PrecisePoint = %v, want present = %v", s.PrecisePoint, tt.wantPoint)
            }
        })
    }
}
```

### Benchmarks

Add benchmarks for performance-sensitive code:

```go
func BenchmarkApplyJitter(b *testing.B) {
    coords := geo.Coordinates{Lat: 40.7128, Lng: -74.0060}
    for i := 0; i < b.N; i++ {
        geo.ApplyJitter(coords, geo.DefaultPrecision)
    }
}
```

Run with: `go test -bench=. -benchmem ./internal/geo/...`

### Writing Component Tests

Use React Testing Library with user-centric queries:

```tsx
import { render, screen, fireEvent } from '@testing-library/react';
import { SearchBar } from './SearchBar';

describe('SearchBar', () => {
  it('renders search input', () => {
    render(<SearchBar onSearch={vi.fn()} />);
    expect(screen.getByRole('searchbox')).toBeInTheDocument();
  });

  it('calls onSearch when form is submitted', () => {
    const onSearch = vi.fn();
    render(<SearchBar onSearch={onSearch} />);

    const input = screen.getByRole('searchbox');
    fireEvent.change(input, { target: { value: 'underground jazz' } });
    fireEvent.submit(input.closest('form')!);

    expect(onSearch).toHaveBeenCalledWith('underground jazz');
  });

  it('has accessible label', () => {
    render(<SearchBar onSearch={vi.fn()} />);
    expect(screen.getByLabelText(/search/i)).toBeInTheDocument();
  });
});
```

### Query Priority

Prefer queries in this order (most to least accessible):

1. `getByRole` — ARIA roles (button, textbox, heading)
2. `getByLabelText` — Form labels
3. `getByPlaceholderText` — Placeholder text
4. `getByText` — Visible text content
5. `getByTestId` — Last resort only

### Mocking API Calls

Use MSW (Mock Service Worker) for API mocking:

```tsx
import { rest } from 'msw';
import { setupServer } from 'msw/node';

const server = setupServer(
  rest.get('/api/scenes', (req, res, ctx) => {
    return res(ctx.json({
      scenes: [
        { id: '1', name: 'Test Scene' },
      ],
    }));
  }),
);

beforeAll(() => server.listen());
afterEach(() => server.resetHandlers());
afterAll(() => server.close());
```

### Accessibility Testing

Use `vitest-axe` for automated accessibility checks:

```tsx
import { axe, toHaveNoViolations } from 'vitest-axe';

expect.extend(toHaveNoViolations);

it('has no accessibility violations', async () => {
  const { container } = render(<SceneCard scene={mockScene} />);
  const results = await axe(container);
  expect(results).toHaveNoViolations();
});
```

Every page component should have an `.a11y.test.tsx` file.

### Testing Zustand Stores

```tsx
import { useAuthStore } from '../stores/authStore';

describe('authStore', () => {
  beforeEach(() => {
    useAuthStore.setState({ token: null, did: null });
  });

  it('sets token on login', () => {
    useAuthStore.getState().login('test-token', 'did:plc:test');
    expect(useAuthStore.getState().token).toBe('test-token');
    expect(useAuthStore.getState().did).toBe('did:plc:test');
  });

  it('clears state on logout', () => {
    useAuthStore.getState().login('test-token', 'did:plc:test');
    useAuthStore.getState().logout();
    expect(useAuthStore.getState().token).toBeNull();
  });
});
```

## E2E Testing (Playwright)

cd e2e && npx playwright install

make test-e2e

# Run with UI
cd e2e && npx playwright test --ui

### Writing E2E Tests

```ts
import { test, expect } from '@playwright/test';

test('user can search for scenes', async ({ page }) => {
  await page.goto('/');
  await page.getByRole('searchbox').fill('underground jazz');
  await page.getByRole('searchbox').press('Enter');
  await expect(page.getByText('Search Results')).toBeVisible();
});
```

### E2E CI Configuration

E2E tests run on PRs that modify `web/`, `e2e/`, or `internal/livekit/`. Failed test artifacts (screenshots, videos) are uploaded for debugging.

## Load Testing (k6)

### Running Load Tests

```bash
make test-load
```

Load test scripts live in `perf/k6/`. They target API endpoints with configurable virtual users and duration.

### Writing k6 Scenarios

```js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '30s', target: 20 },   // Ramp up
    { duration: '1m', target: 20 },     // Steady state
    { duration: '10s', target: 0 },     // Ramp down
  ],
  thresholds: {
    http_req_duration: ['p(95)<300'],    // p95 < 300ms
  },
};

export default function () {
  const res = http.get('http://localhost:8080/api/scenes');
  check(res, {
    'status is 200': (r) => r.status === 200,
    'response time < 300ms': (r) => r.timings.duration < 300,
  });
  sleep(1);
}
```

Integration tests require Docker for a real Postgres database:

```bash

```

These use `testcontainers-go` to spin up ephemeral database containers. They test real SQL queries, migrations, and PostGIS operations.

## Debugging Test Failures

### Go Tests

```bash
# Verbose output with test names
go test -v -run TestFailingTest ./internal/api/...

# Print all log output
go test -v -count=1 ./...  # -count=1 bypasses test cache
```

### Frontend Tests

```bash
# Run specific test with verbose output
cd web && npx vitest run --reporter=verbose src/components/FailingTest.test.tsx

# Debug in browser
cd web && npx vitest --ui
```

### Flaky Tests

If a test passes locally but fails in CI:

1. Run with race detector: `go test -race ./...`
2. Run multiple times: `go test -count=10 ./internal/package/...`
3. Check for shared state between tests (missing `t.Parallel()` or cleanup)
4. Check for timing dependencies (use deterministic values, not `time.Now()`)

## CI Pipeline

### What Runs on Every PR

| Job | Trigger | What It Does |
|-----|---------|-------------|
| `go-test` | All PRs | `go vet` + `go test -race` + coverage → Codecov |
| `frontend-test` | All PRs | `npm run test:coverage` → artifact upload |
| `e2e-tests` | web/, e2e/, livekit changes | Playwright browser tests |
| `accessibility` | Frontend changes | Automated a11y checks |
| `dependency-scan` | All PRs | govulncheck + npm audit |
| `secret-scan` | All PRs | Detect committed secrets |

### Coverage Gates

PRs are blocked if:
- Overall backend coverage drops below 80%
- Overall frontend coverage drops below 70%
- Critical package coverage drops below 95%
- Patch (new code) coverage is below 70%

See `codecov.yml` for the full configuration.
