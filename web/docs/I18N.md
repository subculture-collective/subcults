# Internationalization (i18n) Guide

## Overview

Subcults uses **i18next** and **react-i18next** for internationalization, providing a flexible and maintainable translation system with namespace separation, lazy loading, and automatic language detection.

## Supported Languages

- **English (en)** - Default
- **Spanish (es)** - Placeholder translations provided

## Quick Start

### Using Translations in Components

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation('common');
  return <h1>{t('app.name')}</h1>;
}
```

### Using the Custom Hook

```tsx
import { useT } from '../hooks/useT';

function MyComponent() {
  const { t } = useT('scenes');
  return <h1>{t('title')}</h1>;
}
```

## Namespace Structure

- `common` - Global UI, actions, errors
- `scenes` - Scene management
- `events` - Event management  
- `streaming` - Live streaming
- `auth` - Authentication

## Language Detection

1. User setting (localStorage)
2. Browser language
3. Fallback to English

## Adding Translations

1. Edit `/public/locales/en/{namespace}.json`
2. Add to other locales
3. Run `npm run check:i18n` to validate

## Testing

Missing keys produce console warnings in dev mode.
Translation keys are validated in CI via `check:i18n` script.

See full examples and API reference in the complete documentation.
